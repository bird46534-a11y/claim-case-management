# 理賠案件管理與歸檔追蹤系統

## 系統概述

本系統是一套專為理賠案件管理設計的協作平台，支援多人即時協作、案件狀態追蹤、權限控管與資料導出功能。系統採用簡潔的清單介面與國際印刷風格設計，確保高效率的案件流轉管理。

## 核心功能

### 1. 案件編號管理
- **手動輸入**：使用者新增案件時手動輸入 12 碼案號
- **格式驗證**：自動檢查案號是否符合規定格式
  - 公司代碼（2碼）
  - 區域代碼（2碼）
  - 年度（2碼）
  - 險種（1碼：A/K/M）
  - 流水號（5碼）
- **範例**：`101815A00001`

### 2. 狀態管理
系統提供四個標準化的案件狀態，每次狀態變更都會自動記錄操作人與時間：

| 狀態 | 說明 | 特殊標示 |
|------|------|--------|
| 進入檔案室 | 案件已歸檔 | - |
| 擲回經辦人員 | 案件退回經辦人員處理 | 支援備註原因 |
| 轉台北審核 | 案件轉往台北進行審核 | 橘色標示 |
| 轉法務追償 | 案件移交法務部門追償 | - |

### 3. 即時協作同步
- **WebSocket 連接**：所有在線使用者的畫面即時同步更新
- **無需手動刷新**：新增、編輯、刪除操作立即反映在所有客戶端
- **自動推送**：系統自動廣播所有資料變動

### 4. 權限控管
系統區分兩個使用者角色：

| 角色 | 權限 |
|------|------|
| 管理者/登記員 | 新增案件、編輯狀態、刪除案件、查看歷程 |
| 一般人員 | 搜尋案件、查看狀態、查看歷程 |

### 5. 案件搜尋與時間軸
- **模糊搜尋**：支援案號的快速搜尋
- **歷史時間軸**：點擊案號展開查看完整的狀態變動歷程
- **詳細記錄**：每次變動都記錄操作人、時間與備註

### 6. 資料導出與備份
- **Excel 匯出**：一鍵匯出所有案件資料至 Excel
- **自動備份**：系統每日自動備份資料至 S3 雲端儲存
- **完整歷程**：匯出檔案包含所有案件的完整狀態變動歷程

## 技術架構

### 前端技術棧
- **React 19**：現代化的 UI 框架
- **Tailwind CSS 4**：實用優先的 CSS 框架
- **shadcn/ui**：高品質的 React 元件庫
- **tRPC**：端對端類型安全的 API 通訊
- **Socket.io Client**：即時通訊

### 後端技術棧
- **Express 4**：輕量級 Web 伺服器
- **tRPC**：類型安全的 RPC 框架
- **Drizzle ORM**：現代化的 SQL ORM
- **Socket.io**：即時通訊伺服器
- **ExcelJS**：Excel 檔案生成

### 資料庫結構
- **users 表**：使用者帳戶與角色管理
- **cases 表**：案件基本資訊
- **statusHistory 表**：案件狀態變動歷程

## 使用指南

### 新增案件（管理者）
1. 點擊「新增案件」按鈕
2. 輸入 12 碼案號（例如：`101815A00001`）
3. 系統自動驗證格式
4. 點擊「新增」確認

### 變更案件狀態（管理者）
1. 在案件列表中找到目標案件
2. 點擊「變更狀態」按鈕
3. 從下拉菜單選擇新狀態
4. 若選擇「擲回經辦人員」，需填入擲回原因
5. 點擊「確認變更」

### 查看案件歷程（所有使用者）
1. 點擊案號展開詳細資訊
2. 查看「狀態變動歷程」時間軸
3. 每條記錄顯示：時間、狀態、操作人、備註（如有）

### 搜尋案件（所有使用者）
1. 在搜尋框輸入案號的部分內容
2. 系統自動進行模糊搜尋
3. 結果即時更新顯示

### 匯出資料（所有使用者）
1. 點擊「匯出 Excel」按鈕
2. 系統生成包含所有案件與歷程的 Excel 檔案
3. 自動下載至本地

## 設計理念

### 國際印刷風格
系統採用國際印刷風格（International Typographic Style）設計原則：
- **純白背景**：簡潔清爽的視覺基礎
- **紅色強調**：重要元素與操作按鈕採用紅色突出
- **黑色無襯線字體**：清晰易讀的現代排版
- **非對稱網格**：動態而有序的版面布局
- **細黑分隔線**：精準的視覺層次定義
- **充裕留白**：呼吸感與易讀性

### 使用者體驗
- **即時反饋**：所有操作都有明確的成功/失敗提示
- **權限透明**：清楚顯示使用者可執行的操作
- **無縫協作**：多人同時編輯時自動同步，無衝突
- **易於導出**：一鍵匯出所有資料

## 系統要求

- **瀏覽器**：支援 WebSocket 的現代瀏覽器（Chrome、Firefox、Safari、Edge）
- **網路連接**：穩定的互聯網連接以支援即時同步
- **資料庫**：MySQL 8.0+ 或 TiDB

## 部署與維護

### 環境變數
系統使用以下環境變數（由 Manus 平台自動注入）：
- `DATABASE_URL`：資料庫連接字串
- `JWT_SECRET`：會話加密密鑰
- `VITE_APP_ID`：OAuth 應用 ID
- `OAUTH_SERVER_URL`：OAuth 伺服器地址

### 自動備份
系統每日凌晨 2:00 自動備份所有案件資料至 S3，備份檔案命名格式為 `backup/cases_YYYY-MM-DD.json`。

### 監控與日誌
- 伺服器日誌：記錄所有 API 呼叫與錯誤
- WebSocket 日誌：記錄連接、斷開與訊息傳遞
- 備份日誌：記錄每日備份的執行結果

## 常見問題

**Q：如何重置使用者密碼？**
A：系統使用 Manus OAuth 進行身份驗證，密碼重置由 Manus 平台管理。

**Q：如何備份資料？**
A：系統每日自動備份至 S3，也可手動點擊「匯出 Excel」進行備份。

**Q：支援多少個並發使用者？**
A：系統支援無限並發連接，效能取決於伺服器資源與資料庫配置。

**Q：如何新增自訂狀態？**
A：目前系統僅支援四個預設狀態。若需要自訂狀態，請聯絡系統管理員。
